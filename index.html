Identification ConsoPilote — fiabilité de l'attribution agent — constat d'audit (2026-07-16), depuis élargi (même jour, décision Direction) : toute vente ConsoPilote est désormais commissionnée (visible en Trésorerie/KPI/Mes Ventes), y compris sans ligne Feuil1 correspondante — agent Feuil2 utilisé comme meilleure estimation. Cet onglet reste la référence d'audit : il affiche les 100% des ventes ConsoPilote importées et signale, pour chacune, si l'agent est confirmé par recoupement (Feuil1 + Feuil2 concordants) ou seulement déclaré par une source unique — non affecté par le filtre de période (audit exhaustif).

Méthode : pour chaque vente ConsoPilote, le n° de contrat est recherché dans la colonne « Numéro Contrat ConsoPilote » de Feuil1 (ventes MINT), qui porte l'agent au moment de la vente d'origine ; si le contrat n'y est pas, l'identité du client (téléphone, puis nom exact, puis nom approché ≥90%) est recherchée dans Feuil1 — l'information y existe souvent même sans n° de contrat. Le format sans n° de contrat peut aussi porter une colonne Agent directe (export du 2026-07-17) — « Inconnu » est traité comme non renseigné et laisse jouer le recoupement d'identité ; toute évidence disponible (déclaration directe ou identité) est affichée, l'agent n'est jamais masqué quand au moins une source le fournit. Confirmé = agent concordant entre deux sources (par contrat, par téléphone ou par nom exact) — commissionné normalement. Correspondance approchée = client retrouvé par nom ≥90% seulement — à valider manuellement. Non recoupé / Non identifié = aucune correspondance — l'agent déclaré par la feuille ConsoPilote reste la meilleure estimation et est commissionné (commentaire « agent propriétaire non identifié » conservé pour traçabilité) ; sans date de vente connue, aucune échéance automatique : paiement sur décision manager explicite (Trésorerie). Conflit = sources discordantes — la colonne « Agent retenu » affiche l'agent EFFECTIVEMENT commissionné par le moteur (agent Feuil1 pour un contrat lié Feuil1 ; agent feuille ConsoPilote pour un contrat absent de Feuil1) ; l'écart reste visible ici pour revue manuelle. Source unique Feuil1 = contrat ConsoPilote renseigné dans le suivi MINT mais absent de toutes les feuilles ConsoPilote importées (ex. ZENDA/KOLANI 236103) — commissionné via Feuil1, statut signé/prélèvement à confirmer. Les lignes du nouveau format désignant un client strictement identique à une ligne classique sont dédupliquées (une vente = une ligne, la source la plus riche fait foi).
